Two-Way Binding vs. Local Reference

The example on the left shows how to grab data from the html template, pass it through to properties on the component, and get values back into the template using two-way binding

The right hand example is doing a similiar job, but by using a local reference in the HTML template. It assigns the input to a local reference in the template, and then through an event, sends the informaiton on the whole element through to a method on the component. We can access information on that element as we would any HTML element. This example uses .value since we are using a HTMLInputElement

Takeaway

These are two ways to pass information between your HTML template and the Component TS file. The advantage of two-way binding is that the property of the component is always up to date with the value of the input and visa-versa. The advantage of using a local reference is having access to the entire HTML element.

If you are using the value in the input to update the page real time, for example, a search box which displays possible outcomes for each individual input, two-way binding would be the way to go. On the other hand, if you wanted to change the class or other HTML attributes of the element after an event, you might want to use a local reference.
